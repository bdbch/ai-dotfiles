# Browser Design Review

Open a website in the browser and perform a senior UI/UX design critique, highlighting what works, what doesn't, and actionable improvements.

## When to Use

- Reviewing a new page or feature before launch
- Evaluating visual consistency across the application
- Checking responsive behavior on different viewports
- Identifying usability friction points
- Improving overall design quality

## Review Process

1. **Get the URL** from the user
2. **Open the page** in a fresh browser tab
3. **Take screenshots** at desktop and mobile viewports
4. **Analyze visual hierarchy** — is the most important content prominent?
5. **Evaluate typography** — sizing, line-height, contrast, readability
6. **Check spacing and layout** — consistency, breathing room, alignment
7. **Review color usage** — harmony, contrast, brand alignment
8. **Test interactive states** — hover, focus, active, disabled
9. **Assess responsive behavior** — breakpoints, overflow, touch targets

## Output

A structured critique with:

- **Summary** — overall impression, what's working, what's not
- **Visual Design** — typography, color, spacing, surface treatment
- **Layout & Responsiveness** — grid, breakpoints, density
- **Accessibility** — contrast, focus, touch targets
- **Interaction Design** — feedback, states, micro-interactions
- **Recommendations** — prioritized, actionable improvements
