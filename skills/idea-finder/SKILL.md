# Idea Finder

> Find product improvement ideas — grounded in the actual codebase, structured opportunity analysis.

## When to use

Use this skill when you want product improvement ideas that are grounded in the actual project, not generic brainstorming.

## Before generating

1. Inspect the project: README, package metadata, app structure, docs, APIs, routes, pages
2. Ask about product direction if unclear (target audience, priority, product type)
3. Never produce generic ideas without understanding the product

## Idea categories

- **feature**: New capability that changes the product
- **workflow**: Improved or new user flow
- **ux**: Usability, clarity, feedback, error messages
- **dx**: Developer ergonomics — API naming, config, CLI, error messages
- **api**: Adding/removing/reshaping public API surfaces
- **configuration**: New config options, env vars, flags, defaults
- **onboarding**: First-run experience, getting started
- **discoverability**: Making existing features easier to find
- **docs**: Documentation improvements that change usability
- **trust_safety**: Data handling, permissions, confirmation, undo
- **visual_polish**: Layout, spacing, hierarchy, branding
- **simplification_removal**: Removing features, reducing options
- **performance_perception**: Perceived speed, loading states
- **monetization_activation**: Trial flows, plan gating, upgrade paths

## Output

- Summary: product type, target user, strengths, gaps
- Opportunity areas with ideas per category
- Top 3-5 recommendations with priority, why now, effort
- Quick wins, bolder bets, things to simplify/remove
- Open questions

## Principles
- Think like a senior product strategist, not a software architect
- Ground ideas in the actual product
- Include subtraction ideas — better often means doing less
- Be specific — "improve onboarding" is too vague
- Distinguish novelty and effort levels
- Don't produce false business certainty
