# Crazy Ideas

> Blue-sky ideation — wild, unconventional ideas with zero constraints, no feasibility filtering.

## When to use

Use this skill when you feel stuck in conventional thinking and need a creative jolt — genuinely unexpected ideas, not "improved onboarding."

## Ideation modes

### Adjacent Impossible
Push the product one step past reasonable. What if it was 10x stranger, 10x more ambitious, or served a completely different need?

### Wrong Metaphor
Force the product into an unrelated metaphor. What if this code editor was a musical instrument? What ideas emerge from the comparison?

### Anti-Feature
What would you remove? What if the product did less but felt like more? What if you deleted the core feature?

### Temporal Displacement
Build for a different era. Victorian-era project management? 1980s cyberpunk version? Stone-age developer tool?

### Constraint Inversion
Flip a common constraint. What if there was no save button? No page load? Anonymous users? 5-second API delay?

## Output

- At least 3 ideas, each with: title, one-liner, what_if, why interesting, seed of truth, absurdity level
- Unexpected connections from other domains
- Provocations (deliberately provocative statements)
- Things to steal from other domains
- Bad ideas that might be good
- "Would be cool if" sparks

## Principles
- First 3 ideas are the obvious ones. Push past them.
- Combine things that don't belong together
- Embrace the impractical — extract the principle, discard the impossibility
- Make it weird — if it doesn't make someone uncomfortable, it's not weird enough
- Quantity over quality in raw generation
- Never filter for feasibility — that comes later
